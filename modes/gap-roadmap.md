# Mode: gap-roadmap — 能力缺口 + 优先级行动路径

诚实指出简历改写盖不住的、必须真正去补的能力缺口，并给出可执行的成长方案。

**和 cv-tailor 的分工：** cv-tailor 解决"简历怎么写"，gap-roadmap 解决"我这个人还真缺什么"。

## 核心推理三步

### 第一步：识别真实能力缺口

对比"JD 要的人才画像"和"cv.md 里用户真实具备的能力"，找出所有 gap。

**关键区分——两类 gap 要分开处理：**

| 类型 | 定义 | 处理方式 |
|------|------|----------|
| **包装型 gap** | 用户有相关经历但表达不到位，改写简历就能弥补 | 不列入 gap-roadmap，属于 cv-tailor 的工作范围 |
| **真实型 gap** | 用户确实不具备，改简历盖不住，面试深挖会暴露 | **这才是 gap-roadmap 的对象** |

真实型 gap 的典型特征：如果面试官问"你能现场演示一下 / 你具体怎么实现的 / 你做过类似的项目吗"，用户无法基于真实经历回答。

同时识别用户已有的、可以直接利用的资源：
- cv.md 里正在进行中的项目是否恰好能填补某个缺口？
- 用户已有的某段经历是否只需要加深而不需要从零开始？

如果某个缺口短期内无法补齐（如学历要求、特定行业年限），直接标注"无法短期补齐"，不假装能补，并给出绕过策略。

---

### 第二步：二维优先级打分

对每个真实型 gap，按两个维度评分：

**影响度**（补齐它对拿到这个 offer 的帮助）
- 高：这是 JD 的核心要求，有这个能力面试官明显加分，没有是明显减分
- 中：JD 有提及但非核心，补了有加分，不补影响不大
- 低：JD 边缘涉及，或这类岗位并非必须

**补齐成本**（用户补齐所需的时间精力）
- 低：几小时到一两天可以完成，不需要系统学习
- 中：一到两周的专注投入
- 高：需要数月实践积累，短期内无法真正掌握

**四象限归类：**

| | 补齐成本：低 | 补齐成本：高 |
|--|------------|------------|
| **影响度：高** | 🔥 立刻做（投递前完成） | 📅 规划长线（不影响这次投递，但要启动） |
| **影响度：低** | 🆓 顺手做（有空就做） | ❌ 暂时忽略（收益不值得投入） |

---

### 第三步：生成 3-5 条行动路径

从 🔥 象限开始，按优先级排列行动路径。每条行动必须具体可执行，绝对不写"提升 X 能力"这类空话。

**好的行动描述示例：**
- ❌ 差："了解 Agent 开发"
- ✅ 好："用 Coze 或 Dify 搭一个完整的 Agent 工作流（例如：读取用户输入→调用工具→返回结构化结果），跑通 ReAct 循环和 function calling 的基本逻辑，约半天"

**教程搜索（针对需要学习的行动）：**

对每条涉及学习新知识/技能的 🔥 或 📅 行动，必须用 WebSearch 找到最适合的具体学习资源，不写"自行搜索"。

搜索策略（按质量信号筛选）：
- 中文视频：`"{具体知识点}" 教程 site:bilibili.com` → 取播放量最高且评分≥9.0 的
- 英文视频：`"{具体知识点}" tutorial youtube` → 取百万播放量以上的权威频道
- 文字教程：`"{具体知识点}" tutorial github` → 取 star 数最高的
- 官方文档：直接搜索 `{框架/工具名} official docs` → 官方文档优先于二手整理

在行动路径中直接附上资源（标题 + 链接 + 质量说明），例：
> - 推荐：[Andrej Karpathy《Let's build GPT》](https://youtube.com/...) — YouTube 380万播放，从零手写 Transformer
> - 推荐：[李沐《动手学深度学习》](https://d2l.ai/) — GitHub 4.3万星，配套 B站视频每集20万+

**每条行动包含：**
- 补的是哪个缺口
- 四象限分类（🔥/📅/🆓/❌）
- 建议完成时间线（本周 / 本月 / 长期）
- 具体怎么做（步骤级别，不是方向级别）
- 推荐学习资源（WebSearch 实时找到的，带链接和质量说明）
- 预估耗时

**识别并优先指出"可一石二鸟的行动"：** 如果用户正在做的某个项目恰好能同时补多个缺口，或者某个行动能同时强化简历证明点，要明确标注。

## 输出结构

### 一、能力缺口四象限总览

先用一张表格展示所有识别到的真实型 gap 的分布，让用户一眼看清整体情况。

```
| 缺口描述 | 影响度 | 补齐成本 | 四象限 |
|---------|--------|---------|--------|
| ...     | 高/中/低 | 高/中/低 | 🔥/📅/🆓/❌ |
```

同时列出识别到的"可直接利用的已有资源"（如有）。

---

### 二、优先行动清单

按 🔥 → 📅 → 🆓 顺序输出 3-5 条，每条格式如下：

---

**🔥 行动 N：[行动标题]**

- **补的缺口：** [对应缺口名称]
- **时间线：** 本周 / 本月 / 长期
- **预估耗时：** [X 小时 / X 天]
- **具体做法：**
  1. [步骤 1]
  2. [步骤 2]
  3. ...
- **完成标志：** [怎么判断这件事做完了——可验证的具体产出]
- **推荐资源：** [WebSearch 找到的最佳教程/文档，含链接和质量说明]
- **一石二鸟（如有）：** [如果这个行动同时能强化简历某个证明点，在这里说明]

---

### 三、无法短期补齐的缺口

如果有，单独列出，格式：
- **缺口：** [描述]
- **为什么补不了：** [说明理由]
- **绕过策略：** [如何在面试中处理这个问题，或者用什么替代证明降低影响]

---

## 输出规范

- 全程中文，技术术语（Agent、RAG、LLM、ReAct、function calling、Dify、Coze 等）保留英文
- 四象限总览在前，行动清单在后
- 行动清单每条必须有"完成标志"——这是防止用户做了一半不知道算不算完成的安全锁
- 语气直接，不安慰，不鼓励，不加"你已经很优秀了"这类话——用户要的是真实判断

## 前向指针

分析完成并保存后，在对话末尾输出：

```

- 面试备战（缺口数据会自动带入）→ /nogap interview-prep #{###}
```

若 gap-backlog 有新条目写入，追加：
```
📥 N 个新缺口已记录至 gap-backlog，下次运行 gap-roadmap 时可选择并入。
```

## 保存输出

完整分析保存至 `gap-roadmap/{###}-{公司名拼音或英文}-{YYYY-MM-DD}.md`

- `{###}` = 对应 JDlist 中该 JD 的编号；如本次未关联已有 JD，先执行 JD 协议（见 `_shared.md`）获取编号
- 公司名：小写，空格替换为连字符
- 岗位关键词：JD 岗位名称的 1-2 个核心词，小写，连字符分隔
- 日期：当天

报告头部格式：

```markdown
# 能力缺口分析：{公司} — {岗位}

**关联快速评估：** [链接至 brief/ 对应文件]（或"本次独立运行，无关联评估报告"）
**分析日期：** {YYYY-MM-DD}
**目标岗位类型：** {archetype}

---
```

之后跟完整的三节输出（一、能力缺口四象限总览 / 二、优先行动清单 / 三、无法短期补齐的缺口）。

# Mode: brief — 岗位快速评估（A+B）

收到 JD 时，先执行 JD 协议（`_shared.md` → JD 协议），然后输出 A+B 两个模块并给出评分与推荐行动。

---

## 第 0 步 — 岗位类型识别

将岗位归入以下 6 类之一（详见 `_shared.md`）。若为混合型，标注最接近的 2 类。这决定：
- B 模块优先展示哪些证明点

---

## A 模块 — 岗位概览

输出表格，包含：
- 识别到的岗位类型（archetype）
- 领域（平台/Agent/LLMOps/ML/企业级）
- 职能（开发/咨询/管理/部署）
- 职级
- 远程政策（全远程/混合/现场）
- 团队规模（如 JD 中有提及）
- 一句话概括

---

## B 模块 — 与简历的匹配度

读取 `cv.md`，将 JD 每条要求映射到简历中的具体内容，输出对照表。

**根据岗位类型调整侧重：**
- 前置工程师（FDE）→ 优先展示交付速度和客户对接经验
- 解决方案架构师（SA）→ 优先展示系统设计和集成经验
- 产品经理（PM）→ 优先展示产品挖掘和数据指标
- LLMOps → 优先展示评测、可观测性、pipeline
- Agent 方向 → 优先展示多 Agent 编排、HITL、容错
- AI 转型 → 优先展示变革管理、推广落地、规模化

**差距分析**，每条差距说明：
1. 是硬性门槛还是加分项？
2. 是否有相近经验可以替代？
3. 是否有项目作品能弥补这个空白？
4. 具体应对方案（投递信措辞、快速做一个项目等）

---

## 评估后必做事项

### 1. 执行 JD 协议

见 `_shared.md → JD 协议`：为本次 JD 分配编号并保存至 `JDlist/`。

### 2. 回写分析结论到 JDlist

打开 `JDlist/{###}-*.md`，找到"## 系统分析结论"区块，将占位符替换为本次分析结果：

```
**Archetype：** {识别结果}
**人才画像摘要：** {最看重的 3-5 项能力，逐行列出}
**B 模块主要 gap：** {gap1 / gap2 / gap3（简短描述）}
**评分：** {X/5}
**brief 报告：** [brief/{###}-{公司}-{YYYY-MM-DD}.md]
```

这是下游所有 mode 的共享上下文入口，必须在保存报告前完成。

### 3. 保存快速评估报告

保存至 `brief/{###}-{公司名拼音或英文}-{YYYY-MM-DD}.md`

报告格式：

```markdown
# 快速评估：{公司} — {岗位}

**日期：** {YYYY-MM-DD}
**URL：** {URL 或 未提供}
**岗位类型：** {Archetype}
**评分：** {X/5}
**JD 编号：** #{###}

---

## A) 岗位概览

## B) 与简历的匹配度

---

## 提取的 JD 关键词
（15-20 个关键词，用于简历 ATS 优化）
```

### 4. 记录到 tracker

在 `data/applications.md` 追加一行：
- 顺序编号（与 JD 编号一致）
- 当天日期
- 公司名
- 岗位名
- 评分（1-5）
- 状态：`Evaluated`
- 报告链接（如 `[001](brief/001-company-2026-01-01.md)`）

### 5. 推荐下一步

在报告末尾根据评分和差距情况输出推荐，只推荐最相关的 1-2 个：

```
**下一步推荐：**
- [准备投递简历]  → /nogap cv-tailor #{###}
- [补能力缺口]    → /nogap gap-roadmap #{###}
- [面试前备战]    → /nogap interview-prep #{###}
```

评分低于 3.5 时明确建议不投递，不输出推荐行动。
